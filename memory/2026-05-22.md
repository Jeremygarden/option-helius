### 【核心模型】18指标 Sigmoid + 动量 + 动态Regime权重

#### 评分计算流程（生产版）
```
原始指标值
→ sigmoid_score(v, center, steepness, invert)  # 连续0-100分
→ × momentum_multiplier(当前值, 历史均值, 历史σ, α=0.15)  # 最大±30%
→ × indicator_weight[regime]  # 动态权重
→ Σ 加权求和 → 综合危险分(0-100)
```

Sigmoid公式：`score = floor + (ceil-floor) × σ(-steepness × (v - center))`
- center = 金融先验中性值（非数据拟合）
- invert=True：VIX/PCR等（低值=高危）
- 特例：sectors_200dma 用双sigmoid U形（两端都危险）

#### 统计验证结果（walk-forward v2）
- 全球数据库：37个事件（13美国 + 6欧洲 + 4日本 + 10亚洲新兴 + 2全球）
- 时间跨度：1929-2026
- 训练集：1929-1999（19个事件）；测试集：2000-2026（18个事件）
- 正则化：L2 lambda=0.3，权重边界[2%,12%]，20次随机重启
- 4折扩展窗口CV：1929→1960/1975/1990/2005
- **过拟合比率：1.30x**（v1=2.11x → 改善38%）

### 【三大哨兵】核心业务逻辑

**选择依据**：这3个指标历史上每次大崩盘前都有，且具有明确的先行时间：
1. MOVE > 120 → 债市开始为流动性危机定价（领先3-6个月）
2. HY OAS > 450bps → 信用市场开始担心违约（领先3-6个月）
3. 收益率曲线 < -30bps → 历史最可靠衰退预言（领先12-18个月）

**两哨兵规则**：任意2个同时触发 → 忽略综合分，直接最高警戒
- 历史案例：2007年9月(HY OAS+收益率曲线同时触发) → 2008年崩盘
- 设计原因：综合分可能被"正常"指标拉低，但哨兵指标一旦触发是结构性信号

**接近度计算**：
- MOVE: pct = value/120×100
- HY OAS: pct = value/450×100
- 收益率曲线: pct = max(0, (value-100)/(-30-100)×100)（倒置方向）
